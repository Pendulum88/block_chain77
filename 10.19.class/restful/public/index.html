API : Application Server << 데이터를 주고받는 서버
데이터는 보통 json 형식으로 주고받는다
또는 XML형식(궁금하면 찾아봐)
REST API
REST API : Web 통신에서 클라이언트와 서버가 데이터를 주고 받는 방식 중 하나이다
가장 대표적인 방식이다
REpresentational State Transfer
GET : 데이터를 받을때 사용, 단순 데이터 요청, 파일 등등 / 쿼리스트링 즉, 주소에 출력되는 문자열
POST : 데이터를 보낼때 사용, 저장할떄, 로그인할때, 회원가입할떄, 비밀스러운 정보를 요청할떄 / body 숨겨져서 서버에 데이터를 보낸다
PUT : 데이터를 수정할때 사용, 비밀번호변경, 게시글 수정
DELETE : 데이터를 삭제할때 사용
OPTIONS : 요청을 하기 전에 통신 옵션을 설명하기 위해 사용, 로그인을 하면 => 다른페이지. 이 옵션을 통해서 로그인 정보를 확인하고 그 이후에 GET처리하여 알맞은 데이터(html파일 등)를 응답한다
CORS : 크로스 오리진(Cross-Origin Resource Sharing) / 클라이언트가 하나의 서버에서 HTML파일 등등을 받고 다른 서버에서 미디어(그림,영상 등) 받을때 두 주소가 다르면 원칙적으로 받을수 없게 되어있다. 즉 불펌이 불가능하다. 이걸 허용할수 있게 해주는 기술이 CORS다
PATCH : 데이터를 일부만 수정할때 사용
